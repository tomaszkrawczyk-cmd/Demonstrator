Połączenie z modelem

Bielik nie ma wygodnego hostingu z kluczem API (brak na Groq/OpenRouter/Google). Jedyne realne ścieżki to uruchomienie lokalne (Ollama) albo Hugging Face Inference Endpoints (wymaga samodzielnej konfiguracji serwera). Do demonstracji na żywo poleć gpt-oss lub Gemma.
Jak uruchomić model lokalnie (Windows / PowerShell) 
# 1. Zainstaluj Ollama: https://ollama.com/download
# 2. Pobierz model (raz):
ollama pull gpt-oss:20b
#    albo polski Bielik:
ollama pull SpeakLeash/bielik-11b-v3.0-instruct
# 3. Zezwol przegladarce na dostep (CORS) i uruchom:
$env:OLLAMA_ORIGINS="*"
ollama serve
# 4. W tej aplikacji: srodowisko "Ollama",
#    wpisz nazwe modelu, "Sprawdz polaczenie".
# WAZNE dla RAG: zwieksz okno kontekstu (domyslne 4096 to za malo):
$env:OLLAMA_CONTEXT_LENGTH="16384"
# (ustaw PRZED "ollama serve", w tym samym oknie PowerShell)

gpt-oss-120b (117 mld parametrów) wymaga ~80 GB pamięci — lokalnie nie wstanie na typowym PC; wybierz środowisko OpenRouter, model openai/gpt-oss-120b i podaj klucz API. Uwaga: przez hosting wysyłaj wyłącznie treści jawne.

Korpus aktów prawnych

    Korpus: pełny urzędowy tekst AI Act (artykuły, ustępy, załączniki i wszystkie 180 motywów preambuły — Dz.Urz. UE L 2024/1689, wersja polska) oraz oznaczone streszczenia robocze wytycznych Komisji C(2025) 5052. Asystent rozróżnia rangę źródeł: artykuł to norma, motyw to wykładnia, wytyczne to niewiążąca interpretacja.
    Format importu

    JSON: tablica obiektów {"akt":"AI Act","przepis":"art. 6 ust. 2","tekst":"…","tagi":["…"]}

    TXT: fragmenty rozdzielone linią ===, pierwszy wiersz fragmentu: AKT | PRZEPIS, dalej tekst przepisu.
